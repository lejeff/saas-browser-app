# Life events feature wave

Six sequential tasks polishing the Life Events section and adjacent Real Estate / Assets & Debt categories. Each task ships on its own branch with a manual-test pause and PR/merge cycle before the next begins, per [`workflow.mdc`](../../.cursor/rules/workflow.mdc). The wave builds on the windfalls-as-life-events refactor ([2026-04-28-windfalls-as-life-events.md](./2026-04-28-windfalls-as-life-events.md)) and continues the discriminated-union pattern established by the Real Estate Investment event ([2026-04-25-real-estate-investment-event.md](./2026-04-25-real-estate-investment-event.md)).

## Tasks

1. **Per-holding rental income** — Move `Annual Rental Income` and `Rental income annual appreciation` onto each `RealEstateHolding` so users can attribute rental cash flow to the property that generates it. Each holding compounds its own rental at its own rate. Shipped as PR #25 (`aa701b0`).
2. **Drop global rental from Income & Expenses** — Clean break: remove `rentalIncome` and `rentalIncomeRate` from `PlanInputs` and the form. Legacy stored values leak through `...parsed` in [`storage.ts`](../../app/src/features/planner/storage.ts) but are dropped by the schema (matches the windfall scalar break). Shipped as PR #26 (`e646fc6`).
3. **New Debt life event** — A new `NewDebtEventSchema` variant in `LifeEventSchema` modelling a future loan: principal disburses at `startYear`, amortizes (overTime / inFine) until `endYear`. Mirrors the top-level Debt fields, plus a `startYear` so multiple future loans stack independently. Includes a year-0 disbursement path (when `startYear === currentYear`, the in-loop seeding path is gated on `i > 0` and would skip the loan, so seeding happens pre-loop with no inflator). New-debt schedule summary mirrors the existing top-level `DebtScheduleSummary` text, keyed off the inflated principal so the displayed payment matches what the projection chart shows. Shipped as PR #27 (`232b8c1`).
4. **Color-code Life Event cards** — Each life-event card and its `+ Add` button take the accent color of the category the event flows into, not the Life Events violet: Real Estate Investment → Gold (matches the Real Estate category), Windfall + New Debt → Teal (matches Assets & Debt). The Life Events category header itself stays violet. Pure styling change. Shipped as PR #28 (`425d384`).
5. **Full-pill borders on Liquid / Non-Liquid / Debt subsections** — `CollapsibleSubsection` renders as a `<fieldset>` with the same rounded full-pill border + corner legend treatment as the life-event cards, in place of the old `border-l` left rule. The legend holds a `<button>` containing the title + chevron toggle, so click-to-collapse still works and existing tests that look up subsections via `getByRole("button", { name: /title/i })` keep passing. Pure UI restyle. Shipped as PR #29 (`96641e9`).
6. **`inflateAmount` toggle per life event** — A small inline checkbox (`Adjust amount for inflation`) under the Amount / Purchase amount / Principal field on each life-event card. When checked (default), today's-money inputs are inflated to the landing year (existing convention). When unchecked, the entered amount lands at face value. Helper text always pairs the engine-deposited amount with the relative timing (e.g. `€11,041 in 5 years` checked vs `€10,000 in 5 years` unchecked); falls back to just `in 5 years` when the amount is 0. New-debt schedule summary recomputes against the right principal. The schema field uses `z.boolean().default(true)` so legacy stored events without it hydrate to the historical convention. Shipped as PR #30.

## Cross-cutting decisions

- **Storage round-trip strategy.** Tasks 1–3 introduce new schema fields with sensible zero / default values; Task 6 introduces a new boolean with `.default(true)`. The `EventsSchema.safeParse` and `HoldingsSchema.safeParse` calls in [`storage.ts`](../../app/src/features/planner/storage.ts) both apply schema defaults transparently, so legacy persisted plans hydrate cleanly without a migration step. Tests in [`storage.test.ts`](../../app/src/features/planner/storage.test.ts) explicitly cover each round-trip plus the legacy-without-field case for `inflateAmount`.
- **Year-0 corner case for `NewDebtEvent`.** The main projection loop body is gated on `i > 0`, which means an event with `startYear === currentYear` would silently miss seeding and disbursement. We added a pre-loop block in [`projection.ts`](../../packages/core/src/projection.ts) that handles year-0 seeding (face-value principal, no inflator) so a "loan taken out today" appears in the year-0 projection point. The `inflateAmount` toggle has no effect for this case because the inflator factor is 1 either way; tests in [`projection.test.ts`](../../packages/core/src/projection.test.ts) pin both the year-0 path and the toggle's no-op behavior in that corner.
- **Helper-text consistency.** All three life-event card types (Windfall, Real Estate Investment, New Debt) share the same helper-text contract on their landing-year slider: surface the engine-deposited nominal amount paired with the relative timing when the user has entered something, otherwise just the relative phrase. The card-level `inflatedAmount` / `inflatedPurchaseAmount` / `inflatedPrincipal` accessor toggles between the inflated and face-value branches based on `event.inflateAmount`, so both checkbox states reuse the same render path.
- **Documentation freshness.** Each task updates the matching section in [`docs/architecture.md`](../architecture.md) §4.1 and re-runs `npm run docs:build` so the regenerated PDF + HTML ship in the same commit. The plan archive (this file) is added in the Task 6 PR rather than as a separate post-merge commit (correcting the deviation taken on the windfalls plan, where the archive shipped directly to `main` outside the PR pipeline).

## Ship workflow (per task)

1. Branch from `main` (`feat/<task-slug>`).
2. Implement, run lint + typecheck + test locally, report results.
3. Pause for manual verification in `npm run dev` — the prompt enumerates the specific UI states / inputs / edge cases to exercise.
4. Wait for explicit "ship it" before commit / push / PR.
5. Open PR, watch CI green, squash-merge with `--delete-branch`.
6. `git checkout main && git pull --ff-only origin main`; clean up local branch.
7. Move on to the next task.
